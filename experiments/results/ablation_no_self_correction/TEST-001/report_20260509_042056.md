## Executive Summary

This investigation analyzed a Windows 10 x64 memory image from a suspected APT compromise. The analysis confirms a successful breach with active command and control (C2) infrastructure. Two malicious backdoor processes were identified maintaining persistence on the system: `license_ctrl.e` (PID 1716) and `ftusbsrvc.exe` (PID 4916). Both processes are listening on non-standard ports and the system shows evidence of ongoing communication with external C2 infrastructure at IP address 172.16.4.10 on port 8080. The attack demonstrates sophisticated naming conventions designed to blend in with legitimate system processes.

## Timeline of Events

Based on network connection timestamps and process creation times:

- **2018-09-03 13:52:xx**: System boot and normal service initialization
- **2018-09-04 14:18:xx**: User session establishment and legitimate application startup
- **2018-09-05 14:27:55**: Malicious process `ftusbsrvc.exe` established listener on port 33001
- **2018-09-06 17:08:20**: Additional suspicious process `usbclient.exe` started
- **2018-09-06 17:17:20**: Malicious process `license_ctrl.e` established listener on port 5682
- **Multiple timestamps**: Active C2 communication sessions to 172.16.4.10:8080

## Indicators of Compromise

**Malicious Processes:**
- `license_ctrl.e` (PID 1716) - Backdoor process listening on port 5682
- `ftusbsrvc.exe` (PID 4916) - Backdoor process listening on port 33001

**Network Indicators:**
- C2 Server: 172.16.4.10:8080
- Backdoor Listeners: 0.0.0.0:5682, 0.0.0.0:33001
- Local system IP: 172.16.5.25

**Behavioral Indicators:**
- Processes with deceptive names mimicking legitimate services
- Non-standard port listeners with global binding (0.0.0.0)
- Persistent connections to external infrastructure

## Persistence Mechanisms

The threat actors established persistence through:
1. **Process-based persistence**: Two separate backdoor processes maintaining network listeners
2. **Network-based access**: Multiple listening ports (5682, 33001) allowing remote access
3. **Naming deception**: Process names designed to appear legitimate (`license_ctrl.e` mimicking license management, `ftusbsrvc.exe` mimicking USB services)

## Recommendations

**Immediate Actions:**
1. Isolate the compromised system from the network
2. Block C2 infrastructure (172.16.4.10) at network perimeter
3. Terminate malicious processes (PIDs 1716, 4916)
4. Block ports 5682 and 33001 on network security devices

**Investigation Actions:**
1. Analyze network logs for communication patterns with 172.16.4.10
2. Search for additional systems communicating with the identified C2 infrastructure
3. Examine process execution artifacts to determine initial infection vector
4. Review user account activity around the timeframe of compromise

**Remediation Actions:**
1. Complete system rebuild or comprehensive malware removal
2. Reset credentials for accounts that accessed the compromised system
3. Deploy enhanced monitoring for the identified IOCs across the environment
