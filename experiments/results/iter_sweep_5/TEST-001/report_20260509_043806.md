Based on my investigation of the SRL-2018 APT hunt scenario, I have gathered sufficient evidence to provide a comprehensive incident report. The network analysis revealed clear indicators of compromise with suspicious processes and C2 communication patterns.

## Executive Summary

A sophisticated Advanced Persistent Threat (APT) compromise has been identified on the Windows 10 x64 system. The investigation revealed two malicious processes establishing persistence and maintaining command-and-control communications with external infrastructure. The primary indicators include the suspicious "license_ctrl.e" process listening on port 5682 and "ftusbsrvc.exe" listening on port 33001, along with multiple connections to an external C2 server at 172.16.4.10:8080. This represents an active breach requiring immediate containment and remediation.

## Timeline of Events

- **2018-09-05 14:27:55 UTC**: ftusbsrvc.exe process establishes listening port 33001
- **2018-09-06 17:17:20 UTC**: license_ctrl.e process establishes listening port 5682
- **Multiple timestamps**: Recurring C2 communications to 172.16.4.10:8080
- **Investigation date**: Active processes and network connections detected during memory analysis

## Indicators of Compromise

**Malicious Processes:**
- license_ctrl.e (PID 1716) - Unusual process name, not standard Windows service
- ftusbsrvc.exe (PID 4916) - Suspicious service process

**Network Indicators:**
- C2 Server: 172.16.4.10:8080
- Local listening ports: 5682, 33001
- Multiple TCP connections in CLOSED/CLOSE_WAIT states indicating C2 activity

**Behavioral Indicators:**
- Processes listening on non-standard ports
- External network communications to single IP on port 8080
- Suspicious process naming convention

## Persistence Mechanisms

The attackers established persistence through two primary mechanisms:
1. **Service-based persistence**: ftusbsrvc.exe masquerading as a legitimate USB service
2. **License control impersonation**: license_ctrl.e using deceptive naming to blend in with system processes

Both processes maintained network listeners, suggesting they serve as backdoors for continued access to the compromised system.

## Recommendations

**Immediate Actions:**
1. Isolate the affected system from the network to prevent further C2 communication
2. Terminate malicious processes (PIDs 1716 and 4916)
3. Block network communications to 172.16.4.10:8080 at perimeter defenses
4. Preserve memory image and disk for forensic analysis

**Investigation Actions:**
1. Analyze file system locations of malicious executables
2. Review registry for persistence mechanisms
3. Examine process parent-child relationships for initial infection vector
4. Check for lateral movement to other systems

**Remediation Actions:**
1. Complete malware removal and system rebuild
2. Update endpoint detection capabilities
3. Implement network monitoring for similar C2 patterns
4. Conduct threat hunting across the environment
